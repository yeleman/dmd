Transmission des données

L'envoi des données dans la base de données PNLP se fait à l'aide d'un masque de saisie Excel.

  • Ce masque de saisie est unique et permet de saisir l'ensemble des indicateurs (hors DHIS).
    Pour des raisons de commidités, il est disponible sous la forme d'un fichier par DPS.
  • Le masque de saisie comprend une ligne pour les données concernant la DPS elle-même puis une ligne pour chaque ZS.
  • Il n'est pas nécessaire de remplir tous les champs. Seuls ceux remplis seront pris en compte. Lors de l'envoi, un récapitulatif permettra de vérifier que les données transmises ont été prises enregistrées.
  • Les cellules  noires  indiquent qu'aucune donnée n'est attendue à ce niveau pour cet indicateur.

Période

  • Ce masque doit être utilisé et transmis chaque fois que vous obtennez les données d'un indicateur s'y trouvant. Ce peut être chaque mois pour les données de routine ou occasionnellement pour les données d'enquêtes.
  • Les cellules  jaune  permettent de définir la période qui s'applique aux données (l'année et le mois).
    Ce champ jaune a pour seul but de simplifier la saisie en recopiant cette période dans les lignes suivantes.
  • La période est indispensable. Chaque ligne doit donc contenir une période valide.
    Pour les enquêtes de longue durée, la période est celle de la collecte des informations.
  • Vous pouvez bien sûr envoyer des données pour des périodes différentes au sein d'un même fichier.
  • Pour saisir un même indicateur sur une même localité (ZS, DPS) mais pour deux périodes différentes, il faut dupliquer la ligne concernée et modifier la période. cf. Modifications ci-dessous.

Modification

  • Pour limiter les erreurs, le fichier est verrouillé. Vous pouvez le déverrouiller avec le mot de passe PNLP.
  • Ne modifez pas les noms du DPS ou des ZS dans le fichier. Cela empêcherai l'envoi de fonctionner.
